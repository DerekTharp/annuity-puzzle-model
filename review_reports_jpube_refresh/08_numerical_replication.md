# Reviewer 8: Numerical Methods and Replication

## 1. Summary Assessment
The project is materially stronger than the version implied by the stale manuscript. The biggest upgrade is that the decomposition is now backed by full subset enumeration and an exact Shapley calculation, so the headline 7-channel result of 41.4% to 18.3% and the 9-channel full-model result of 6.6262% are not just one-path artifacts. That said, the numerical case is still not fully clean: quadrature sensitivity remains sizable, Euler residuals are only moderate, and the manuscript is still out of sync with the refreshed outputs.

## 2. Strongest Advances
- The decomposition is now genuinely order-independent in the authoritative path. `run_subset_enumeration.jl` enumerates all 512 subsets and computes exact Shapley values from that lookup table, which is a real credibility gain over a single sequential ordering. See [run_subset_enumeration.jl](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/scripts/run_subset_enumeration.jl#L416) and [subset_enumeration.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/subset_enumeration.csv#L513).
- The exact Shapley output is internally consistent. The channel contributions in [shapley_exact.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/shapley_exact.csv#L1) sum to the total drop between the empty set and the full 9-channel model, which is exactly what I want to see in a replication-grade decomposition.
- The new decomposition makes the source of the 9-channel drop transparent. Most of the additional reduction from 18.3% to 6.6262% comes from `Age needs` and the correlated-risk channels, while `State utility` is numerically tiny. That is a useful identification result, not just a table filler.
- The project now has a real validation stack, not just a narrative one: grid checks, quadrature checks, Euler residuals, moment validation, and counterfactuals are all being generated from code rather than hand-entered text.

## 3. Main Weaknesses
- The manuscript is still materially stale relative to the data products. [main.tex](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/paper/main.tex#L46), [main.tex](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/paper/main.tex#L73), [main.tex](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/paper/main.tex#L75), and [main.tex](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/paper/main.tex#L448) still quote the older 5.3%, 23.9%, and 46.6% figures, while the refreshed tables report 18.33%, 36.08%, and 57.62% in [welfare_counterfactuals.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/welfare_counterfactuals.csv#L2). That mismatch will trigger immediate credibility questions.
- Quadrature convergence is still the main numerical risk. [convergence_diagnostics.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/convergence_diagnostics.csv#L3) through [convergence_diagnostics.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/convergence_diagnostics.csv#L6) show ownership moving from 27.92% at 5 nodes to 17.84% at 11 nodes, with the production 9-node value at 21.38% and the 120x50, 11-node reference at 16.78%. That is not catastrophic, but it is not “knife-edge-free” either.
- The Euler diagnostics are acceptable for a policy paper, but not especially sharp. [euler_residuals.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/euler_residuals.csv#L2) reports a mean residual of 1.37%, 10.07% of interior points above 1%, 5.99% above 5%, and a maximum residual of 0.9915. The appendix already acknowledges the piecewise-linear interpolation floor, but the residual profile still leaves room for concern. See [appendix.tex](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/paper/appendix.tex#L412).
- The codebase still contains a Monte Carlo Shapley path. [run_shapley_decomposition.jl](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/scripts/run_shapley_decomposition.jl#L39) uses only 200 random permutations, and the file itself says so. That is fine as a legacy or exploratory tool, but it is a reproducibility trap unless it is clearly deprecated, because the authoritative exact result lives elsewhere.
- The moment validation is a sanity check, not a full external validation. [run_moment_validation.jl](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/scripts/run_moment_validation.jl#L72) starts from one representative fair-health household with $250,000 of wealth, and the comparison moments in [moment_validation.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/moment_validation.csv#L7) and [moment_validation.csv](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/tables/csv/moment_validation.csv#L17) show medical spending running above the cited targets at most ages. That is useful, but it is not yet a strong out-of-sample fit test.
- The simulation pipeline uses averaged policies under medical shocks rather than a fully shock-contingent policy in forward paths. See [solve.jl](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/src/solve.jl#L198) and [simulation.jl](/Users/derektharp/Documents/_Projects/26.29%20Annuity%20Puzzle%20Simulation/src/simulation.jl#L76). That approximation is defensible, but it should be stated more explicitly because it affects how much weight to put on the simulated moments.

## 4. What Must Improve Before Submission
- Reconcile the manuscript, appendix, and tables with the refreshed outputs. The current paper text still reads like an older version of the project.
- Either tighten the quadrature benchmark further or present a more disciplined robustness argument for why 9 nodes are sufficient despite the 3 to 11 node spread.
- Push the Euler diagnostics lower, or explicitly frame them as a coarse but adequate approximation for a piecewise-linear solver.
- Make the exact subset-enumeration script the single source of truth for Shapley results, and mark the 200-permutation script as deprecated.
- Upgrade the validation section so it is not just one simulated path from one starting state. At minimum, show sample-level validation or a richer set of external moments.

## 5. Score
6.5/10.

## 6. Venue Recommendation
JPubE is still plausible, but only as a revision-driven submission. The computational architecture is now meaningfully stronger because the project has exact subset enumeration and exact Shapley accounting, which is a real step up in replication credibility. I would not call the numerical evidence fully submission-ready yet, though, because the manuscript is stale, quadrature sensitivity is still material, and the validation section is more suggestive than definitive. If those are cleaned up, this can be a credible JPubE paper; in the current state, it is a promising but not yet fully de-risked JPubE submission.
